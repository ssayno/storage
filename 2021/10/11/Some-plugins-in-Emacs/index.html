Some plugins in Emacs

Posted on 2021-10-11 Edited on 2021-10-14 In Emacs , magit
Symbols count in article: 2.6k Reading time ≈ 2 mins.

Preface

This blog is used to introduce some plugins in Emacs

magit

Before use magit in Emacs, you should know how to use git in terminal.

git

If you want some information about the git, you can read this book.
Magit is an interface to the version control system Git, implemented as an Emacs package.

Basic useage

Now, the best way is use clone to get a repository from github or gitee, and then the next steps are pretty simple.

  • Check the status, use C-x g to get the same result of git status in terminal.(Maybe better)

  • s can be used to stage. Unstaged changes to Staged changes(git add ….)

  • Put the cursor on the Staged changes and print c to see all the options(most of them I never use, maybe I am a chicken dish), and then also print c to come true the function of git commit, if you finish, use C-c C-c to finish it

  • Put the cursor on the Unmerged ...., and print P(uppercase), and print p to come true the function of git push.

  • F(uppercase) can come true the function of git pull

Add remote repo to local repo

For me, when use git in terminal, use command git add remote origin <url> not create new branch in remote repo, but if I use this command in Emacs by magit, it will create new branch.(Maybe I can’t understand it)

Others about branch

  • You can use C-x g you can to show git status(If there is no git repository, it will remind you to create one)

  • It will show all untracked files, for example

  • Then you can use this buffer, and move cursor to the Untracked file and enter the s to stage untracked files.(You also can enter the u to change the Staged changes to Unstracked files)

  • When you put the cursor on the Staged changes and print d you can see all options, then you can print u to see the difference unstaged.

  • Putting the cursor on the Staged changes is same as above.

  • commit staged file
    Also put the cursor on the Staged changes and print c all the options you can choose, for me, use c to use commit, if you finish you can use C-c C-c to done it.

  • show log history
    put the cursor on the Head and print l then you can see all options, and if you use it, you can use q to quit.

  • move change to stash
    Moving the cursor to the Unstaged changes and print z you can see all the options, in Stash you can choose to finish this(Others I don’t know about it so far), and input some message, them you can become it to stash(state)

  • move a stash(If you delete, it will dieappear)
    you can use z to show options, and print p to delete a stash.

  • apply a stash(If you apply the stash, it will keep in status and can usenext time.)

  • create new branch
